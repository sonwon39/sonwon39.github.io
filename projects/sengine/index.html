그래픽스 논문을 직접 구현하며 만든 DirectX 12 기반 실시간 렌더링·시뮬레이션 엔진입니다.

rendering pipeline부터 resource 버퍼 생성 및 관리와 같은 엔진 코드를 처음부터 작성했고, 그 위에서 렌더링, 시뮬레이션, 그리고 컴포넌트 기반 게임엔진까지 세 트랙으로 발전시켰습니다.

렌더링

PBR · IBL

시뮬레이션

Stable Fluids, Curl Noise

엔진 코어

Actor · Component

렌더링

물리 기반 렌더링 (PBR)

Cook-Torrance BRDF(GGX distribution, Smith visibility, Schlick Fresnel)와 metallic·roughness 워크플로를 구현했습니다. 여기에 환경 맵과 BRDF LUT를 이용한 IBL(image-based lighting)을 추가해 주변 환경맵에 의한 조명까지 반영했습니다.

시뮬레이션

Stable Fluids 유체 시뮬레이션

마우스를 이용해 밀도장과 속도장을 추가해준 뒤 semi-Lagrangian advection과 pressure projection으로 비압축 조건을 만족시키는 Stable Fluids를 GPU 컴퓨트 셰이더로 구현했습니다. 여기에 vorticity confinement를 추가해, 수치 소산으로 사라진 소용돌이 디테일을 되살렸습니다. 최종적으로 밀도장을 RGB 색으로 렌더링해주었습니다.

Curl Noise 입자 유동

Perlin noise를 해시 기반으로 평가해 fBm(fractal Brownian motion)을 쌓고, 그 포텐셜장의 curl을 취해 발산이 0인 비압축성 속도장을 만들고, 이 속도장을 따라 입자를 이류시키면, 속도장을 직접 풀지 않고도 유체 같은 소용돌이 흐름을 얻을 수 있습니다.

엔진 코어

// 액터 = 컴포넌트의 합성
auto player = std::make_shared<Actor>("Player");
auto mesh   = std::make_shared<SkinnedMeshComponent>(player.get());
auto cam    = std::make_shared<CameraComponent>(player.get());
auto col    = std::make_shared<CollisionComponent>(player.get());

player->SetRootComponent(mesh); // 루트는 메시
mesh->Attach(cam);              // 카메라를 자식으로 (3인칭)
mesh->Attach(col);              // 콜리전도 자식으로

player->OnRegister();           // 트리 전체에 등록 시점 알림

Actor / Component 기반 씬 구성

언리얼 엔진의 Actor 모델을 참고해, 씬에 배치되는 모든 오브젝트를 Actor와 여러 ActorComponent의 합성으로 표현합니다. Actor 자체는 트랜스폼을 갖지 않고 루트 SceneComponent에 위임하며, 기능이 필요할 때마다 컴포넌트를 하나 붙이는 방식으로 확장됩니다.

컴포넌트는 책임이 점진적으로 늘어나는 계층으로 설계했습니다. ActorComponent(소유 Actor + Tick 인터페이스)에서 출발해, SceneComponent가 localTransform과 부모/자식 계층을 더하고, PrimitiveComponent가 가시성·PSO/텍스처·per-primitive constant buffer·PhysX 트랜스폼 동기화를 얹습니다. 그 위에 StaticMesh / SkinnedMesh / Camera / Light / Collision Component가 구체 역할을 맡습니다.

트랜스폼은 localTransform(부모 기준)을 단일 원본으로 두고, 부모의 월드 트랜스폼과 합쳐 worldMatrix를 산출합니다. 변경 시 자식들에게 자동 전파되어, 계층 어디서 트랜스폼을 바꿔도 일관된 월드 좌표가 유지됩니다.
